# 파일 기반 e-Commerce (콘솔 쇼핑몰)

KOPO 2026 자바 최종과제로 만든 콘솔 쇼핑몰입니다. JSON 파일을 DB처럼 사용하고, 로그인한 계정의 권한(관리자 / 일반회원)에 따라 메뉴가 나뉩니다.

과제 조건에 "코드 개발에 ChatGPT 등 LLM을 참조하지 말 것"이 명시돼 있어서, 구조 설계부터 로직까지 전부 직접 작성했습니다.

<img src="docs/assignment/page-01.png" width="640" alt="표지">

## 실행 방법

- 실행 클래스: `Ecommerce`
- 관리자 로그인은 이메일 `admin`, 비밀번호 `admin`.
- 일반회원은 회원가입 후 관리자가 승인해야 로그인됩니다. 가입 직후 상태는 `가입요청`입니다.
- 실행하면 `src/data/` 아래 JSON 파일들을 읽어옵니다.

JDK 21 / Gradle 9 환경에서 `gradlew build`로 빌드됩니다.

## 사용 기술

Java 21, Gradle, Jackson(`ObjectMapper`)을 썼습니다. 별도 DB 없이 JSON 파일 6개로 데이터를 저장하고, 객체와 JSON 사이 변환은 전부 Jackson으로 처리했습니다.

## 폴더 구조

과제에서 지정한 패키지 구조를 그대로 따랐습니다.

```
kr.co.javaex.sec23
├─ Ecommerce.java     실행 진입점
├─ controller         콘솔 메뉴 / 입력 처리
├─ service            비즈니스 로직
├─ repository         JSON 파일 읽기·쓰기 (DB 역할)
├─ domain             데이터 클래스(POJO)
└─ util               JsonUtil(Jackson), InputUtil, enum 상태값
```

계층 구조와 데이터 흐름을 다이어그램으로 정리했습니다.

## 기능

관리자

- 카테고리: 등록 / 수정 / 삭제, 대분류·중분류(자기참조), 정렬 순서
- 상품: 등록 / 수정 / 삭제 / 조회, 재고 수정, 판매중지, 재고가 0이면 자동 품절
- 상품-카테고리 매핑: 등록 / 수정 / 삭제 / 조회
- 회원 관리: 전체 조회, 가입 승인, 권한 변경, 탈퇴 처리

일반회원

- 회원: 가입(형식 검증) / 로그인 / 로그아웃 / 정보 수정 / 비밀번호 변경 / 탈퇴 요청
- 상품 보기: 전체, 카테고리별, 가격순, 이름 검색, 상세
- 장바구니: 담기 / 조회 / 수량 변경 / 삭제 / 비우기
- 주문: 장바구니 전체·선택 주문, 바로 주문, 내역·상세 조회, 취소(재고 복구)

## 실행 화면

직접 실행한 콘솔 출력입니다. (반복되는 메뉴 출력은 일부 줄였습니다.)

**로그인 → 권한별 메뉴 분기**

```text
==============================
1. 로그인
2. 회원가입
0. 종료
==============================
메뉴 선택 >> 1
========== 로그인 ==========
이메일 입력: admin
비밀번호 입력: admin
관리자님 로그인되었습니다.

===== 관리자 메뉴 =====
1. 카테고리 관리
2. 상품 관리
3. 상품-카테고리 매핑 관리
4. 주문 관리
5. 회원 관리
6. 로그아웃
0. 이전
```

**카테고리 대분류 · 중분류 트리**

```text
===== 카테고리 목록 =====
전자제품 (C001, 정렬순서: 1)
  └ 노트북 (C101, 정렬순서: 1)
  └ 모니터 (C102, 정렬순서: 2)
사무용품 (D004, 정렬순서: 1)
  └ 볼펜 (D401, 정렬순서: 1)
생활용품 (C002, 정렬순서: 2)
  └ 주방용품 (C201, 정렬순서: 1)
  └ 청소용품 (C202, 정렬순서: 2)
  └ 욕실용품 (C203, 정렬순서: 3)
인간 (E001, 정렬순서: 4)
  └ 이상혁 (E101, 정렬순서: 1)
```

**회원 승인 · 권한 변경 (관리자)**

```text
선택 >> 2
승인할 회원ID 입력: qwer4405
회원 승인 완료

선택 >> 4
권한 변경할 회원ID 입력: qwer4405
1. 일반회원
2. 관리자
변경할 권한 선택: 2
권한 변경 완료

선택 >> 1
===== 전체 회원 목록 =====
회원ID : qwer4405
회원명 : 김민혁
상태 : 정상
권한 : 관리자
------------------------
(그 외 회원 생략)
```

**주문 → 재고 차감 → 취소 → 재고 복구** — 핵심 로직

주문 전, P003(삼성 노트북) 재고는 5입니다.

```text
===== 상품 상세 정보 =====
상품ID : P003
상품명 : 삼성 노트북
가격 : 1300000
재고수량 : 5
상태 : 정상
```

2개를 바로 주문하면 주문이 생성되고 재고가 차감됩니다.

```text
선택 >> 3
주문할 상품 ID 입력: P003
주문 수량 입력: 2
상품 주문이 완료되었습니다.

===== 내 주문 목록 =====
주문번호 : O008
상품명 : 삼성 노트북
수량 : 2
주문금액 : 2600000
```

주문 후 다시 상세를 보면 재고가 5 → 3으로 줄어 있습니다.

```text
===== 상품 상세 정보 =====
상품ID : P003
재고수량 : 3
상태 : 정상
```

주문을 취소하면 재고가 다시 복구됩니다 (3 → 5).

```text
선택 >> 6
취소할 주문번호 입력: O008
주문 취소가 완료되었습니다.

===== 상품 상세 정보 =====
상품ID : P003
재고수량 : 5
상태 : 정상
```

## 구현하면서 신경 쓴 점

- 콘솔이라 컨트롤러를 새로 만들면 로그인 상태가 사라집니다. 그래서 `UserService`를 `main`에서 한 번만 만들어 컨트롤러에 넘겨주는 방식으로 로그인 정보를 유지했습니다.
- 주문하면 재고를 깎고, 취소하면 되돌립니다. 재고가 0이 되면 자동으로 품절로 바꿉니다. 주문 처리 중에는 검증을 모두 통과한 다음 한 번에 파일로 저장하도록 했습니다.
- 회원과 상품 상태는 enum으로 관리했습니다. 회원은 `가입요청 → 정상 → 탈퇴요청 → 탈퇴완료`, 상품은 `정상 / 판매중지 / 품절`.
- 회원 ID와 비밀번호는 정규식으로 형식을 검사하고, 숫자·양수·필수 입력 같은 처리는 `InputUtil`에 모아 뒀습니다.

## 데이터 파일

- `user.json` : 회원
- `categories.json` : 카테고리(대·중분류)
- `products.json` : 상품
- `product-category-mapping.json` : 상품-카테고리 매핑
- `carts.json` : 장바구니
- `orders.json` : 주문

## 앞으로 (2단계)

지금은 JSON 파일에 저장하지만, 다음에는 JDBC로 실제 DB를 붙이고 최종적으로 Spring Boot로 옮길 생각입니다. 과제에서 지정한 `controller / service / repository` 구조가 스프링과 거의 같아서 옮기는 부담은 크지 않을 것 같습니다. 자세한 건 학교 시험과 과제를 끝낸 뒤에 정리할 예정입니다.
